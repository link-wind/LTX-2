# 训练主线文字图

> 这张图的目标不是把 `ltx-audio-trainer` 的每个实现细节都展开，而是把“文生音频训练到底是怎么串起来的”压成一张可以反复查看的总图。如果你现在最想搞清楚的是：原始数据要长什么样、预处理之后会变成什么、训练时到底喂给 transformer 的是什么，这一页就是给这个问题准备的。

---

## 总流程图

```text
┌─────────────────────────────────────────────────────────────┐
│                        RAW DATASET                          │
│                                                             │
│  Caption                                                    │
│  Audio Path                                                 │
│                                                             │
│  Example metadata row:                                      │
│  {"caption": "...", "audio_path": "audios/0001.wav"}        │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                    OFFLINE PREPROCESSING                    │
│                                                             │
│  Caption -> Gemma Text Encoder + Embeddings Processor       │
│          -> conditions/*.pt                                 │
│                                                             │
│  Audio Waveform -> Audio VAE Encoder                        │
│                -> audio_latents/*.pt                        │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                  TRAINING DATA LOADING                      │
│                                                             │
│  PrecomputedDataset                                         │
│  + DataLoader                                               │
│                                                             │
│  audio_latents/                                             │
│  conditions/                                                │
│                                                             │
│  Match rule: each latent file must have a paired            │
│  condition file                                             │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                    TRAINING STEP SETUP                      │
│                                                             │
│  latent_conditions["latents"]                              │
│  -> AudioPatchifier.patchify()                              │
│                                                             │
│  text_conditions                                            │
│  -> audio_prompt_embeds / video_prompt_embeds               │
│  -> prompt_attention_mask                                   │
│  -> optional text conditioning dropout                      │
│                                                             │
│  timestep_sampler.sample_for(audio_latents)                 │
│  -> sigma                                                   │
│  sample noise                                               │
│  build noisy_audio                                          │
│  build positions                                            │
│  build timesteps                                            │
│  build loss_mask                                            │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                    MODEL INPUT ASSEMBLY                     │
│                                                             │
│  clean latent                                               │
│  noise                                                      │
│  sigma                                                      │
│  noisy latent = (1 - sigma) * clean + sigma * noise        │
│  positions                                                  │
│  timesteps                                                  │
│  context                                                    │
│  context_mask                                               │
│                                                             │
│  -> audio Modality                                          │
│  -> ModelInputs                                             │
│     ├── audio                                               │
│     ├── audio_targets = noise - clean                       │
│     └── audio_loss_mask                                     │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                     TRANSFORMER FORWARD                     │
│                                                             │
│  audio Modality -> transformer -> audio_pred                │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                      LOSS COMPUTATION                       │
│                                                             │
│  (audio_pred - audio_targets)^2                             │
│  + apply audio_loss_mask                                    │
│  -> masked MSE                                              │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                 VALIDATION / CHECKPOINTING                  │
│                                                             │
│  validation prompts -> generate wav samples                 │
│  optional W&B audio logging                                 │
│  save model/lora checkpoint                                 │
│  save training_state for resume                             │
└─────────────────────────────────────────────────────────────┘
```

## 压缩成一句话

```text
原始 caption 和音频
-> 预处理成 text feature / audio latent
-> 训练时把 clean audio latent patchify 后加噪
-> 再补上 positions、timesteps、context、mask
-> 组装成 audio Modality / ModelInputs
-> 送进 transformer
-> 预测 target = noise - clean
-> 用 masked MSE 训练
```

---

## 分层看这条主线

### 第一层：原始数据层

```text
caption
audio_path
```

这一层还只是训练样本的“描述信息”。

`ltx-audio-trainer` 当前默认的原始数据入口是一份：

```text
csv / json / jsonl metadata file
```

其中每一行至少要有：

```text
caption
audio_path
```

---

### 第二层：预处理产物层

```text
conditions/
audio_latents/
```

这一步由两个离线脚本完成：

```text
caption
-> text encoder.encode()
-> embeddings_processor.process_hidden_states()
-> save conditions/*.pt

audio waveform
-> encode_audio(...)
-> save audio_latents/*.pt
```

对应的总入口是：

```text
process_audio_dataset.py
```

它会分别调：

```text
process_captions.py
process_audio.py
```

---

### 第三层：训练读取层

```text
PrecomputedDataset
```

训练时不再读原始 wav 和 caption，而是直接读预处理好的 `.pt` 文件。

读取时最关键的是“配对关系”：

```text
audio_latents/xxx.pt
<-> conditions/xxx.pt
```

如果某个 latent 找不到对应 condition，这条样本就不会进入有效训练集。

---

### 第四层：训练输入组装层

这一层发生在：

```text
TextToAudioStrategy.prepare_training_inputs()
```

它会做几件核心的事：

```text
1. 取出 clean audio latent
2. patchify
3. 采样 sigma
4. 采样 noise
5. 构造 noisy latent
6. 取出 text context / context_mask
7. 构造 positions
8. 构造 timesteps
9. 构造 loss mask
10. 组装成 audio Modality 和 ModelInputs
```

---

### 第五层：模型与监督层

```text
audio Modality
-> transformer
-> audio_pred
-> compare with audio_targets = noise - clean
-> masked MSE
```

也就是说，这个 trainer 学的不是“直接回归原始波形”，而是：

```text
在 latent 空间里预测一个 velocity-style target
```

---

## 原始数据到训练的文件流

### 原始数据文件

```text
dataset.jsonl / dataset.csv
audios/0001.wav
audios/0002.wav
...
```

例如：

```text
{"caption": "soft piano with female vocal", "audio_path": "audios/0001.wav"}
{"caption": "dark cinematic drone", "audio_path": "audios/0002.wav"}
```

---

### 预处理后的目录

```text
.precomputed/
├── conditions/
│   ├── audios/0001.pt
│   └── audios/0002.pt
└── audio_latents/
    ├── audios/0001.pt
    └── audios/0002.pt
```

其中：

```text
conditions/*.pt
-> text embeddings + attention mask

audio_latents/*.pt
-> audio latent tensor + shape metadata
```

---

## 普通 text-to-audio 路径

```text
caption + audio
-> preprocess to conditions + audio_latents
-> dataloader reads paired .pt files
-> patchify audio latent
-> sample sigma
-> sample noise
-> build noisy audio latent
-> build positions / timesteps / context / context_mask
-> build audio Modality
-> transformer
-> audio_pred
-> compare with audio_targets = noise - clean
```

这条路径里没有 video branch，也没有 reference branch。

所以它的核心就是：

```text
单一音频 latent 分支
+ 单一文本条件分支
```

## 关键词关系图

```text
clean audio latent
├── + noise
├── + sigma
└── -> noisy audio latent

text_conditions
├── -> audio_prompt_embeds
├── -> video_prompt_embeds (fallback)
└── -> prompt_attention_mask

noisy audio latent
├── + positions
├── + timesteps
├── + context
├── + context_mask
└── -> audio Modality

audio Modality
└── -> transformer

target
└── noise - clean audio latent

loss mask
└── decides which tokens contribute to loss
```

---

## 训练时最关键的张量关系

```text
audio_latents
-> patchify
-> [B, T, D]

sigma
-> [B]

noise
-> same shape as patchified audio_latents

noisy_audio
-> (1 - sigma) * clean + sigma * noise

targets
-> noise - clean

timesteps
-> sigma broadcast to [B, T]

positions
-> patch positions generated by AudioPatchifier

context
-> text embeddings

context_mask
-> prompt attention mask
```

## validation / checkpoint 支线

训练主线之外，这个 trainer 还有两条重要支线。

### validation 支线

```text
validation.prompts
-> cache embeddings (optional)
-> ValidationSampler.generate()
-> wav samples
-> optional wandb.Audio logging
```

### checkpoint 支线

```text
transformer weights / lora weights
-> save safetensors

global_step + rng + scheduler + optimizer(optional)
-> save training_state_step_xxxxx.pt
```

也就是说，训练过程中不只是“算 loss”，还会周期性地：

```text
生成验证音频
保存 checkpoint
保存 resume state
```

---

## 最后再压缩一次

如果把 `ltx-audio-trainer` 的训练过程只保留最核心的一根线，那么就是：

```text
原始 caption + wav
-> 预处理成 conditions + audio_latents
-> dataloader 读 paired .pt
-> patchify clean audio latent
-> sample sigma + noise
-> build noisy latent / positions / timesteps / context / masks
-> audio Modality
-> transformer forward
-> predict noise - clean
-> masked MSE
-> validation / checkpoint / resume
```
